# CSC EUDI Wallet Remote Signing Plugtest 2026 – Event Playbook Overview

## Purpose of this Playbook
This playbook provides the operational framework for the CSC EUDI Wallet Remote Signing Plugtest 2026.
Its purpose is to ensure that all participants clearly understand the technical scope of the interoperability tests, the roles and components provided by each participant, the testing scenarios executed during the event, the technical endpoints and credentials required, and the process for coordinating and validating interoperability.

The playbook serves as a shared operational reference for all teams during the event, enabling participants to quickly identify integration partners, understand available services, and execute interoperability scenarios in a structured way.

The framework encourages collaboration, technical exchange, and real integration between independent implementations. Successful interoperability depends not only on the technical setup but also on active communication between participants.

## Overview of the Event
The plugtest brings together ecosystem participants involved in the European Digital Identity Wallet (EUDI Wallet) remote signing architecture, with a focus on implementations of the Cloud Signature Consortium (CSC) API v2.2.
The event validates that independent implementations across the ecosystem can interoperate successfully in real-world signing scenarios.

Participants may provide components such as wallets, relying parties or verifiers, remote signing services (CSC servers), QTSP trust services, credential issuers, and supporting infrastructure.

The objective is to validate that wallets can successfully invoke remote signing services, CSC APIs behave consistently across implementations, credentials can be issued and used across systems, verifiers can validate signatures generated by remote signing services, and certificate trust chains can be processed correctly across implementations.

## Collaboration Between Participants
A key principle of the plugtest is direct collaboration between participants.
Before beginning any interoperability test, teams are expected to communicate with the participants they intend to test with. This discussion should confirm the roles and components involved, verify API endpoints and environments, confirm authentication methods and credentials, clarify supported operations and capabilities, and agree on the specific scenario being executed.

These discussions ensure that both sides understand how their systems interact and allow participants to resolve integration issues quickly. The event encourages open technical discussion between teams throughout the testing sessions.

## Plugtest Coordination Platform
To support interoperability testing, the event provides a central coordination platform.
The platform acts as a technical registry of all participants and their components. It allows teams to discover integration partners and access the technical information required for testing.

### Participant Directory
The platform contains a directory of all participating organizations involved in the event.
For each participant, the directory provides organization name, technical roles, contact points during the event, and an overview of the technical components available for testing.

### Participant Technical Pages
Each participant has a dedicated technical page within the platform.
These pages act as the primary technical reference for interoperability testing and contain integration details such as:
* **Roles and capabilities** (Wallet, CSC Remote Signing Service, Verifier, Issuer, QTSP)
* **Technical endpoints** (CSC APIs, verification endpoints, wallet integration endpoints)
* **Authentication and access information** (API tokens, credentials, test accounts)
* **Credential information** (credential templates, identifiers, certificate chains, trust anchors)
* **Signing capabilities** (signHash, signDoc, supported algorithms and formats)
* **Integration notes** (environment constraints, example requests, limitations)

## Testing Workflow
Testing during the event follows a collaborative workflow:
1. Identify another participant to test with
2. Review the participant’s technical page
3. Discuss integration details with the partner team
4. Configure endpoints and credentials
5. Execute the agreed interoperability scenario
6. Validate the results together
7. Record the outcome in the shared results matrix

Participants are encouraged to test with multiple partners to maximize interoperability coverage.

## Interoperability Results Matrix
The event maintains a shared interoperability results matrix.
The matrix provides a real-time overview of interoperability results across participants. It records which integrations have been successfully completed, which combinations of components have been tested, and where issues may exist.

This shared visibility allows participants to quickly identify additional testing opportunities and areas requiring further validation.

## CSC API v2.2 Discussion and Improvement Feedback
An important objective of the plugtest is to collect practical feedback on the CSC API v2.2 specification.
During interoperability testing, participants are encouraged to discuss and document any areas where the APIs could be improved. This includes observations about implementation challenges, unclear specification details, inconsistencies between implementations, and potential improvements to the API structure.

Feedback may relate to:
* API clarity and specification wording
* Endpoint behaviour and interoperability issues
* Authentication and authorization flows
* Credential lifecycle operations
* Signing operations such as `signHash` and `signDoc`
* Error handling and response formats
* Implementation complexity or ambiguity

Participants should discuss these findings directly with other teams during testing sessions and record improvement suggestions where appropriate.
The goal is to ensure that the plugtest not only validates interoperability but also contributes practical input that can help evolve and strengthen future versions of the CSC specification.

## Expected Outcomes
By the end of the plugtest, the event aims to achieve successful cross-vendor remote signing integrations, identify implementation differences, improve CSC API interoperability, collect practical ecosystem feedback, and strengthen collaboration between participants.

The results and feedback generated during the event contribute directly to maturing the EUDI Wallet remote signing ecosystem and supporting future deployments.
